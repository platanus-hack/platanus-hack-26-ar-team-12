<objective>
Hacer que Beto se vea como un asistente serio para adultos mayores: la burbuja muestra 5 estados visuales claros (idle / listening / thinking / speaking / error) con anillo de color, ícono central simple, animación gentle pulse, y transiciones de ≤200ms. `BetoTheme` define tokens centralizados (tipografía ≥22sp, colores high-contrast WCAG AA) que toda UI propia (Compañero, sheets, atajos) consume — sin sizes locales.
</objective>

<user_story>
Como adulto mayor, quiero ver de un vistazo qué está haciendo Beto sin tener que esperar la voz, y quiero que toda la información que me muestra Beto sea grande, contrastada y fácil de leer.
</user_story>

<must_haves>
- OVERLAY-05: 5 estados con paleta fija — idle gris, listening azul, thinking ámbar, speaking verde, error rojo. Transiciones ≤200ms via `ObjectAnimator`.
- Logo de Beto **se mantiene estable** en el centro de la burbuja en todos los estados. Solo cambia el anillo de color + ícono pequeño superpuesto + animación.
- Anillo: `View` separado con `bubble_state_ring.xml` selector que cambia según `BubbleState`.
- Animaciones: `idle` sin animación, `listening` pulse suave (alpha 0.6 ↔ 1.0, 800ms), `thinking` pulse + slight scale (0.95 ↔ 1.05), `speaking` pulse rápido (400ms), `error` shake (translation X ±8dp, 300ms, 2 ciclos).
- UX-01: `BetoTheme.kt` define `textStyleHero = 28sp` y `textStyleBody = 22sp` con altura de línea ≥1.4. Compose `Typography` para Compañero + Material `TextAppearance` para Views legacy si hace falta.
- UX-02: Mensajes de error siempre desde `PhraseFallbacks.UNKNOWN_COMMAND`-style — nunca exponer códigos.
- UX-03: TTS feedback sigue limitado a 1-2 frases máximas (heurística en `PhraseGenerator` — Plan 04-01).
- Colors high-contrast: contraste ≥4.5:1 sobre fondo blanco/gris para texto; ≥3:1 para anillo de estado sobre fondo de la app target.
</must_haves>

<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| Animación tartamudea en device de demo (frames drop) | medium | Animaciones simples (ObjectAnimator native, no Compose dentro de Service). Tests visuales en device antes de demo. |
| Color del anillo invisible sobre fondos similares (verde sobre WhatsApp) | medium | Anillo tiene borde negro 1dp para legibilidad sobre cualquier fondo. |
| Estados se quedan colgados (ej. listening eterno si STT no devuelve) | medium | `BubbleState.LISTENING` tiene timeout 30s en `OverlayManager` que vuelve a `IDLE` con TTS warm error si no recibió `VoiceCaptured`. |
| Sixth state se cuela accidentalmente | low | `BubbleState` es enum cerrado de 5 valores. Nuevos estados rechazados en code review. |
| Overlap de eventos (listening + speaking) | medium | Máquina de estados explícita en `OverlayManager` con transiciones legales documentadas. |
</threat_model>

<title>OverlayBubble.applyState() + animaciones</title>
<action>
`OverlayBubble.kt`:

```kotlin
fun applyState(state: BubbleState) {
    if (currentState == state) return
    currentState = state

    // Color del anillo
    ringView.backgroundTintList = ColorStateList.valueOf(
        ContextCompat.getColor(context, state.ringColor)
    )

    // Ícono pequeño
    if (state.centerIcon != 0) {
        stateIconView.setImageResource(state.centerIcon)
        stateIconView.visibility = View.VISIBLE
    } else {
        stateIconView.visibility = View.GONE
    }

    // Animación
    runningAnimator?.cancel()
    runningAnimator = when (state.animation) {
        BubbleAnimation.NONE -> null
        BubbleAnimation.PULSE_SLOW -> pulseAnimator(800)
        BubbleAnimation.PULSE_SCALE -> pulseScaleAnimator(800)
        BubbleAnimation.PULSE_FAST -> pulseAnimator(400)
        BubbleAnimation.SHAKE -> shakeAnimator()
    }?.also { it.start() }
}

private fun pulseAnimator(durationMs: Long): ObjectAnimator =
    ObjectAnimator.ofFloat(ringView, "alpha", 0.6f, 1.0f).apply {
        duration = durationMs
        repeatCount = ObjectAnimator.INFINITE
        repeatMode = ObjectAnimator.REVERSE
        interpolator = AccelerateDecelerateInterpolator()
    }
```

Latencia de transición ≤200ms — para PULSE / SHAKE el efecto es continuo, no hay transición visible per se.

<action>
`OverlayManager`:

```kotlin
class OverlayManager(/*...*/) {
    private var currentState: BubbleState = BubbleState.IDLE
    private var listeningTimeoutJob: Job? = null

    init {
        AgentBus.events.onEach { handleEvent(it) }.launchIn(scope)
    }

    private fun handleEvent(event: AgentEvent) {
        val next = when (event) {
            is AgentEvent.VoiceCaptureStarted -> BubbleState.LISTENING
            is AgentEvent.VoiceCaptured -> BubbleState.THINKING
            is AgentEvent.IntentLaunched -> BubbleState.SPEAKING
            is AgentEvent.TtsCompleted -> BubbleState.IDLE
            is AgentEvent.ToolFailed -> BubbleState.ERROR
            is AgentEvent.VoiceCaptureTimeout -> BubbleState.ERROR
            else -> currentState
        }
        if (next != currentState) {
            transitionTo(next)
        }
    }

    private fun transitionTo(state: BubbleState) {
        currentState = state
        bubble.applyState(state)
        Timber.tag("Beto-Overlay").d("BUBBLE_STATE_CHANGED to=$state")

        // Auto-recovery
        when (state) {
            BubbleState.LISTENING -> startListeningTimeout()
            BubbleState.ERROR -> scheduleErrorClear()
            else -> listeningTimeoutJob?.cancel()
        }
    }

    private fun startListeningTimeout() {
        listeningTimeoutJob = scope.launch {
            delay(30_000)
            AgentBus.emit(AgentEvent.VoiceCaptureTimeout)
        }
    }

    private fun scheduleErrorClear() {
        scope.launch { delay(1500); transitionTo(BubbleState.IDLE) }
    }
}
```

- `TtsCompleted`
