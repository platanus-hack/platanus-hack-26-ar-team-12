# Phase 4 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-phase, verified inline because GSD UI agents are not installed in this runtime.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Android Views for floating overlay; Compose-style tokens may be used for Companion/shortcut UI if Compose exists from Phase 3 |
| Icon library | Android vector drawables in `res/drawable`; simple state icons only |
| Font | Android system default; no custom font dependency in hackathon scope |

**Implementation surface:** the floating bubble remains a classic Android View/WindowManager overlay. Do not rebuild it in Compose. Shared UI tokens live in Kotlin code, not XML-only styling, so both overlay and any Beto-owned screen can consume the same semantic values.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Bubble ring inset, icon inset, compact gaps |
| sm | 8dp | Bubble internal padding, inline UI spacing |
| md | 16dp | Default Beto-owned screen padding |
| lg | 24dp | Senior UI section padding |
| xl | 32dp | Large touch target spacing |
| 2xl | 48dp | Major vertical separation in Companion/shortcut UI |
| 3xl | 64dp | Bubble diameter and page-level spacing |

Exceptions:

- Bubble diameter is fixed at `64dp`.
- Bubble ring stroke/inset may use `4dp`.
- Close target from Phase 1 may keep existing `88dp` size because it is already a large drag target.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 22sp | 400 | 1.35 |
| Label | 22sp | 600 | 1.25 |
| Heading | 28sp | 700 | 1.20 |
| Display | 32sp | 700 | 1.15 |

**Required tokens:**

- `textStyleBody`: minimum `22sp`
- `textStyleHero`: minimum `28sp`

All Beto-owned Companion/shortcut UI must use these tokens. No screen-local text size below `22sp` is allowed for user-facing text, except system status bar/notification text controlled by Android.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Beto-owned UI background / inner bubble surface |
| Secondary (30%) | `#111827` | Primary readable text on light surfaces |
| Accent (10%) | state-specific | Bubble state ring and state icon only |
| Destructive | `#D32F2F` | Error state ring/icon and close target active state |

Accent reserved for:

- Bubble state ring
- Bubble state icon
- Focus/active affordance in Beto-owned UI

Do not use accent colors for all interactive text. Text must remain high contrast.

### Bubble State Palette

| State | Ring | Icon | Animation |
|-------|------|------|-----------|
| idle | `#6B7280` gray | neutral dot or home/rest icon | none |
| listening | `#1565C0` blue | microphone | shared gentle pulse |
| thinking | `#F59E0B` amber | simple dots or clock | shared gentle pulse |
| speaking | `#2E7D32` green | speaker | shared gentle pulse |
| error | `#D32F2F` red | exclamation | shake |

Fallback/agentic work reuses `thinking`. No sixth visual state is allowed.

### Contrast Rules

- Text/background contrast must be high contrast by token choice and manually verified on the demo phone.
- Ring/icon contrast must remain distinguishable against the white inner bubble.
- Error red must never be used for routine fallback work; fallback is thinking amber.

## Interaction Contract

### Floating Bubble

- Size: `64dp x 64dp`.
- Identity: stable `project_logo.png` inside the circle.
- State expression: color ring + simple icon.
- No text labels or badges inside the 64dp bubble.
- State transition duration: maximum `200ms` end-to-end for color/icon/animation state changes.
- Non-idle animation: one shared gentle pulse.
- Error animation: shake only, then return to idle or the next real state.

### Agentic Fallback Feedback

- When fallback starts, Beto speaks exactly: `Dame un segundo, lo intento de otra forma.`
- Bubble moves to `thinking` while fallback runs.
- If 3 contact/search attempts fail, leave WhatsApp as-is, speak warm failure, and return bubble to idle.
- Do not show or speak technical errors.
- Do not visually imply the message was sent; success means the message is filled in the compose field.

### Senior UI

- Beto-owned screens must prioritize readable type, simple hierarchy, and large touch targets.
- Do not add settings panels, dense controls, or small utility text in this phase.
- Manual verification on the demo phone is part of the definition of done.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Fallback starting | Dame un segundo, lo intento de otra forma. |
| Contact clarification | Como lo tenes agendado? |
| Contact not found after attempts | No pude encontrarlo. Probemos de nuevo despues, dale. |
| Generic warm failure | Uy, algo no salio bien. Probemos de nuevo, dale. |
| Message prepared success | Listo, te deje el mensaje preparado. |
| Error state visible text | No pude hacerlo ahora. Probemos de nuevo, dale. |
| Destructive confirmation | Not applicable — Phase 4 must not auto-send or perform destructive actions. |

Rules:

- Fallback/failure TTS is one sentence maximum.
- Contact clarification may be one concise question.
- No technical terms in user-facing copy: no "Intent", "LLM", "Accessibility", "JSON", "timeout", "exception", "node", or error codes.
- Spanish tone: warm, brief, Argentine, simple, not childish.

## Verification Contract

Automated checks:

- `BubbleState` defines exactly `idle`, `listening`, `thinking`, `speaking`, `error`.
- Bubble transition duration constant is `<= 200L`.
- State palette contains the five locked colors above.
- `BetoTheme` contains `textStyleHero` with `28.sp` or greater and `textStyleBody` with `22.sp` or greater.
- `PhraseBank` contains the exact fallback and failure phrases from this spec.
- No user-facing phrase in `PhraseBank` contains technical words listed in the copywriting rules.

Manual demo-phone checks:

- Bubble states are distinguishable on the physical phone.
- Non-idle pulse is visible but not distracting.
- Error shake is brief and does not loop.
- Body text in Beto-owned UI is readable at normal viewing distance.
- Contrast is acceptable under venue lighting.
