# Plan 04-01: Agentic Contracts, Tree Serialization, and Safety Primitives

<objective>
Create the local data contracts and deterministic safety primitives that make the Phase 4 agentic fallback plannable and testable before any real Accessibility action or Gemini call is wired. This plan defines the one-action decision schema, compact tree snapshot format, per-snapshot node refs, tree hashing, stuck detection, and bus events used by later integration.
</objective>

<user_story>
As a Beto developer, I need the agentic loop to operate through strict local contracts, so that Gemini can suggest one bounded action without directly controlling the phone.
</user_story>

<must_haves>
- D-05: Allowed agentic actions are exactly click, type, scroll, back, and abort.
- D-06: Agentic hard-limit constants exist as `MAX_ITERATIONS = 5`, `TIMEOUT_MS = 15000L`, and `MAX_TOKENS = 4000`.
- D-07: Stuck detection aborts on unchanged tree hash or repeated action/ref.
- D-08: The model may include a short internal plan, but the executable decision contains exactly one action.
- D-12: No persistent alias/contact learning is introduced.
- AI-SPEC: Runtime execution is local Kotlin; model output is a structured `AgenticDecision`.
- Research: Tree serialization is the context-control mechanism and must cap to 50 nodes.
</must_haves>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Model emits broad or destructive action | high | `AgenticAction` enum contains only `CLICK`, `TYPE`, `SCROLL`, `BACK`, `ABORT`; no `SEND` action exists anywhere in the schema. |
| Stale node ref acts on wrong UI element | high | `NodeRefRegistry` refs are per-snapshot only; Plan 04-02 must refresh live nodes before `performAction`. |
| Loop repeats forever | high | `StuckDetector` catches unchanged tree hash and repeated action/ref before integration. |
| Sensitive screen text is sent to cloud/logs | medium | `TreeSerializer` redacts DNI/phone/card-like strings before serialized output leaves the service boundary. |
| Executor drops context decisions | medium | Every D-05 through D-08 primitive is represented as concrete constants/tests in this plan. |
</threat_model>

- `CLICK`, `SCROLL`, `BACK`, and `ABORT` must reject non-null `text`.
- No enum value or validation branch may contain `SEND`.

- `TreeHash.kt`
  - expose deterministic hash of normalized snapshot text
  - ignore volatile ref numbers only if implementation can preserve stable semantics; otherwise hash serialized text directly
