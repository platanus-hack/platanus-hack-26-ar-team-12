# Plan 04-03: Bubble States and Senior UX Contract Implementation

<objective>
Implement the Phase 4 UI contract: five bubble states with stable logo, semantic ring colors, simple icons, shared pulse/error shake, 200ms transitions, shared senior typography/color tokens, and an exact phrase bank for warm non-technical feedback.
</objective>

<user_story>
As an older adult, I want Beto's bubble and own UI to clearly show what it is doing, use readable text, and speak short warm phrases, so that I feel guided instead of confused.
</user_story>

<must_haves>
- D-13: Bubble uses stable logo plus color ring and simple icon; no labels/badges inside 64dp bubble.
- D-14: Palette is idle gray, listening blue, thinking amber, speaking green, error red.
- D-15: Non-idle states use one shared gentle pulse; error uses shake.
- D-16: Agentic fallback reuses thinking; no sixth state.
- D-17: State transitions complete within 200ms.
- D-18: Define `textStyleHero >= 28sp` and `textStyleBody >= 22sp`.
- D-19: Beto-owned Companion/shortcut UI must use shared tokens, not local sizes.
- D-20: Exact phrase bank exists for common status/failure cases.
- D-21: Phrase bank is warm and brief.
- D-22: High-contrast color tokens exist and must be manually checked on demo phone.
- D-23: Fallback/failure TTS is one sentence; contact clarification can be one concise question.
- UI-SPEC: Bubble stays 64dp and state transition duration is max 200ms.
</must_haves>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| User cannot tell whether Beto is listening/thinking/speaking/error | medium | Five-state semantic color + icon mapping with tests and manual demo-phone check. |
| UI looks too small for older adults | medium | Shared `BetoTheme` tokens enforce 22sp body and 28sp hero minimums. |
| Error copy exposes technical terms | medium | `PhraseBank` owns copy; tests block technical words like Intent, LLM, Accessibility, JSON, timeout, exception, node. |
| Fallback appears as alarming error state | medium | Agentic fallback uses thinking amber, not red error. |
| Transition lag breaks roadmap criterion | medium | Bubble transition duration constant is <= 200ms and tested. |
</threat_model>
