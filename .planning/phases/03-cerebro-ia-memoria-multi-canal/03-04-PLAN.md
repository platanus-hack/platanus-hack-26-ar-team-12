<objective>
Implementar la memoria persistente del usuario como un único JSON (`user_memory_v1`) guardado en `EncryptedSharedPreferences`. El modelo `UserMemory` contiene aliases de contactos (mapping `"nieto" → ContactRef`), preferencias de canal por contacto, y datos personales declarados por el usuario (familia, hobbies, ciudad, etc.). Operaciones thread-safe vía `Mutex`. La memoria es la fuente de verdad para los clarificadores en Plan 03-05 — si Beto ya sabe quién es "nieto" o por qué medio le manda a Juan, no pregunta.
</objective>

<user_story>
Como adulto mayor, no quiero que Beto me pregunte dos veces lo mismo. Si ya le dije que mi nieto es Juan, debe recordarlo para siempre. Si ya le dije que a Juan le mando por WhatsApp, debe recordarlo también.
</user_story>

<must_haves>
- MEM-01: `UserMemory` data class serializable con `aliases: Map<String, ContactRef>`, `channelPreferences: Map<ContactRef, Channel>`, `profile: Map<String, List<String>>` (key = categoría, value = facts), `version: Int = 1`.
- MEM-02: `UserMemoryStore` guarda `UserMemory` serializado JSON en `EncryptedSharedPreferences` con clave `user_memory_v1`. Lectura cacheada en memoria; escritura siempre persiste.
- MEM-03: API consultas: `knowsAlias(alias)`, `resolveAlias(alias)`, `preferredChannel(contact)`. No suspend (consultas síncronas con cache en memoria).
- MEM-04: API mutaciones: `recordAlias`, `recordChannel`, `recordFact(category, fact)`. Suspend con `Mutex` para garantizar thread-safety.
- Reset/clear API expuesta para testing y para botón "olvidar todo" en debug builds.
- Migración future-safe: campo `version` permite agregar campos sin romper builds viejos.
</must_haves>

<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| Datos personales en SharedPreferences plain text | high | `EncryptedSharedPreferences` con `MasterKey` AES-256 GCM. |
| Lecturas concurrentes corrompen el JSON | medium | `Mutex` envuelve toda escritura; lecturas son del cache en memoria. |
| Escritura interrumpida deja el JSON corrupto | medium | `EncryptedSharedPreferences.edit().commit()` es atómico. Si parsing falla al boot, log + reset a `UserMemory.empty()`. |
| Crecimiento ilimitado del JSON (memoria de Compañero, etc.) | low | Phase 3 escribe poco (aliases + canales + facts cortos). Phase 4 Compañero NO persiste historial por default. |
| Sensitive data en logs | medium | NUNCA loguear contenido del JSON. Solo `MEMORY_UPDATED key=alias` o conteos. |
</threat_model>

<title>Modelo UserMemory + Channel + ContactRef</title>
<action>
`memory/UserMemory.kt`:

```kotlin
@Serializable
data class UserMemory(
    val version: Int = 1,
    val aliases: Map<String, ContactRef> = emptyMap(),
    val channelPreferences: Map<String, Channel> = emptyMap(), // key = ContactRef.id (Long.toString)
    val profile: Map<String, List<String>> = emptyMap(),
) {
    fun knowsAlias(alias: String) = aliases.containsKey(alias.lowercase())
    fun resolveAlias(alias: String) = aliases[alias.lowercase()]
    fun preferredChannel(contact: ContactRef) = channelPreferences[contact.id.toString()]

    fun withAlias(alias: String, contact: ContactRef): UserMemory =
        copy(aliases = aliases + (alias.lowercase() to contact))

    fun withChannelPreference(contact: ContactRef, channel: Channel): UserMemory =
        copy(channelPreferences = channelPreferences + (contact.id.toString() to channel))

    fun withFact(category: String, fact: String): UserMemory =
        copy(profile = profile + (category to (profile[category].orEmpty() + fact)))

    companion object {
        fun empty() = UserMemory()
    }
}

@Serializable
data class ContactRef(val id: Long, val displayName: String, val phoneE164: String)

@Serializable
enum class Channel { WHATSAPP, SMS, CALL }
```

Notar: `ContactRef` es snapshot — si el contacto cambia número en la libreta, Plan 03-05 reconcilia con `ContactRepository`.

<task id="03-04-T05" type="implement">
<title>UserMemoryStore con EncryptedSharedPreferences</title>
<action>
`memory/UserMemoryStore.kt`:

```kotlin
class UserMemoryStore(context: Context) {

    private val prefs: SharedPreferences by lazy {
        val masterKey = MasterKey.Builder(context)
            .setKeyScheme(MasterKey.KeyScheme.AES256_GCM)
            .build()
        EncryptedSharedPreferences.create(
            context,
            "beto_user_memory",
            masterKey,
            EncryptedSharedPreferences.PrefKeyEncryptionScheme.AES256_SIV,
            EncryptedSharedPreferences.PrefValueEncryptionScheme.AES256_GCM,
        )
    }

    private val mutex = Mutex()
    private val json = Json { ignoreUnknownKeys = false; encodeDefaults = true }

    private val _state = MutableStateFlow(load())
    val state: StateFlow<UserMemory> = _state.asStateFlow()

    private fun load(): UserMemory = runCatching {
        prefs.getString("user_memory_v1", null)?.let { json.decodeFromString<UserMemory>(it) }
            ?: UserMemory.empty()
    }.getOrElse {
        Timber.tag("Beto-Memory").w(it, "Corrupted memory, resetting")
        UserMemory.empty()
    }

    suspend fun recordAlias(alias: String, contact: ContactRef) = mutex.withLock {
        update { it.withAlias(alias, contact) }
    }

    suspend fun recordChannel(contact: ContactRef, channel: Channel) = mutex.withLock {
        update { it.withChannelPreference(contact, channel) }
    }

    suspend fun recordFact(category: String, fact: String) = mutex.withLock {
        update { it.withFact(category, fact) }
    }

    suspend fun clear() = mutex.withLock {
        _state.value = UserMemory.empty()
        prefs.edit().remove("user_memory_v1").apply()
    }

    private fun update(transform: (UserMemory) -> UserMemory) {
        val next = transform(_state.value)
        prefs.edit().putString("user_memory_v1", json.encodeToString(next)).apply()
        _state.value = next
        Timber.tag("Beto-Memory").d("MEMORY_UPDATED")
    }

    // Lecturas síncronas
    fun current(): UserMemory = _state.value
}
```

Inicializar singleton en `BetoApplication.onCreate()`:
```kotlin
override fun onCreate() {
    super.onCreate()
    // ...
    Container.userMemoryStore = UserMemoryStore(this)
}
```
