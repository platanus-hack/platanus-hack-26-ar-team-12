<objective>
Implementar el cliente Gemini 2.5 Flash vía Firebase AI Logic con structured tool calling, system prompt argentino con few-shots, sanitizer regex aplicado en el límite (LlmClient) antes de cualquier payload, y cache por hash para resiliencia ante red caída en demo. El cliente expone una API suspend `interpret(transcript): Decision` donde `Decision` es un sealed type con `ToolCall` (acción a ejecutar), `NeedsClarification` (falta info — qué preguntar), o `Unknown` (fallback al matcher).
</objective>

<user_story>
Como dev de Beto, necesito un cliente Gemini estable y type-safe que reciba un transcript en es-AR y devuelva una decisión estructurada (tool call o pregunta), de modo que el resto del sistema (router, clarificadores, dispatcher) pueda integrarse sin pelearse con strings ni JSON suelto.
</user_story>

<must_haves>
- LLM-01: Interface `LlmClient` con `GeminiLlmClient` default + `AnthropicLlmClient` fallback comentado.
- LLM-02: `PromptBuilder` con system prompt en es-AR, few-shots argentinos cubriendo los 4 canales (WhatsApp, SMS, llamada, Maps), tool descriptions en español.
- LLM-03: Tool calling con `temperature: 0`, allow-list de tool names, validación estricta del JSON, retry 1x si malformado.
- LLM-05: Cache LRU en memoria por hash SHA-256 del input sanitizado (TTL 30 min, max 50 entradas).
- PRIV-01: `Sanitizer` con regex que tachan DNI argentino (7-8 dígitos), teléfonos AR (`\+?54\s?\d{10}` o `\d{10,11}`), tarjetas (16 dígitos con/sin espacios o guiones).
- PRIV-02: Sanitizer aplicado **dentro** de `GeminiLlmClient.generateContent()`, no en callers — garantía estructural.
- AI-SPEC: Output del LLM es un `Decision` sealed type validado con `kotlinx.serialization` (`ignoreUnknownKeys=false`).
</must_haves>

<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| API key Gemini leak (key embebida en APK) | high | App Check con Firebase es opcional en hackathon (PROJECT.md). Demo controlada — aceptamos riesgo. Documentar en `LlmClient.kt`. |
| LLM emite tool no soportada o args malformados | high | Allow-list de `tool_name` en `Decision.deserialize`; retry 1x con error inline; si retry falla → `Decision.Unknown` y caller cae al matcher. |
| Payload con DNI/tel/tarjeta sale a Gemini | high | `Sanitizer` aplicado dentro del cliente. Tests unitarios cubren los 3 patrones. |
| Cache devuelve respuesta stale en demo (red volvió) | low | TTL 30 min — alcanza para demo. Operación de invalidación expuesta para emergencia. |
| Falta de red durante demo bloquea el flujo | medium | Caller puede detectar `IOException` y caer al matcher determinista (Plan C). LLM no es bloqueante para el guion. |
| LLM emite respuesta de >4000 tokens (costo + latencia) | medium | `maxOutputTokens=512` por config. |
</threat_model>

<title>Implementar Sanitizer</title>
<action>
Crear `android/app/src/main/java/com/beto/app/llm/Sanitizer.kt`:

```kotlin
object Sanitizer {
    private val DNI = Regex("""\b\d{7,8}\b""")
    private val PHONE = Regex("""(\+?54\s?)?(\d[\s-]?){10,11}\d""")
    private val CARD = Regex("""\b(\d[\s-]?){15}\d\b""")

    fun sanitize(input: String): String =
        input
            .replace(CARD, "[TARJETA]")
            .replace(PHONE, "[TEL]")
            .replace(DNI, "[DNI]")
}
```

Notas:
- Orden importa: tarjeta primero (16 dígitos), luego teléfono (10-11), luego DNI (7-8) para evitar que tarjeta matchee como DNI.
- Tests deberían pasar después de este task.

`GeminiLlmClient.kt`:

```kotlin
class GeminiLlmClient(
    private val cache: LlmCache,
) : LlmClient {
    private val model: GenerativeModel = Firebase.ai(backend = GenerativeBackend.googleAI())
        .generativeModel(
            modelName = "gemini-2.5-flash",
            generationConfig = generationConfig {
                temperature = 0f
                maxOutputTokens = 512
                responseMimeType = "application/json"
            },
        )

    override suspend fun interpret(rawTranscript: String): Decision {
        val sanitized = Sanitizer.sanitize(rawTranscript)
        cache.get(sanitized)?.let { return it }

        val prompt = PromptBuilder.buildInterpretPrompt(sanitized)
        val response = runCatching { model.generateContent(prompt).text.orEmpty() }
            .getOrElse { return Decision.Unknown }

        val decision = parseDecision(response) ?: parseDecision(retryOnce(prompt, response))
            ?: Decision.Unknown

        cache.put(sanitized, decision)
        return decision
    }
    // ... parseDecision con allow-list, retryOnce con error inline
}
```

Comentado al final del archivo: stub `AnthropicLlmClient` (no compila) como pista del fallback.
