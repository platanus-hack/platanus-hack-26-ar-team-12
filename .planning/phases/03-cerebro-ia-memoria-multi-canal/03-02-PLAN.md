<objective>
Mejorar la robustez del STT en es-AR con dos palancas independientes pero complementarias: (1) preferir el recognizer **on-device** (`createOnDeviceSpeechRecognizer`, API 31+) cuando el dispositivo lo soporta — mejor para acentos y voces senior, sin red; (2) un `SttCorrector` que toma el `rawTranscript` cuando es ambiguo (confianza baja o no matchea Plan C) y lo manda a Gemini con contexto (lista de contactos del usuario, último comando) para que devuelva una versión corregida. El corrector NO se invoca para todos los inputs — solo cuando hace falta.
</objective>

<user_story>
Como adulto mayor con acento marcado o voz suave, quiero que Beto entienda lo que digo aunque mi micrófono o mi pronunciación no sean perfectas, sin que tenga que repetir tres veces.
</user_story>

<must_haves>
- STT-FIX-01: `SttCorrector.correct(raw, context)` invoca Gemini con prompt corto y devuelve transcript corregido. Context incluye lista de nombres de contactos del usuario (de `ContactRepository`) y, si existe, el último comando ejecutado.
- STT-FIX-02: `RecognizerFactory.create()` devuelve `OnDeviceRecognizer` si `SpeechRecognizer.isOnDeviceRecognitionAvailable()` y la versión del paquete on-device para `es-AR` está disponible; sino `CloudRecognizer` (el actual de Phase 2).
- El corrector se aplica **solo si**: (a) el recognizer reporta `confidence < 0.6`, **o** (b) `DeterministicMatcher` no matchea Y el transcript tiene < 8 palabras (típico comando corto). Si la sentence es larga (Modo Compañero futuro), no se corrige automáticamente.
- El corrector tiene timeout 1.5s; si vence, se devuelve el `rawTranscript` sin corregir (degradación elegante).
- El corrector NO modifica nombres propios que no estén en `context.knownContacts` — para evitar que invente.
</must_haves>

<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| Corrector altera intención del usuario (ej. cambia "mandale" por "llamale") | high | Prompt explícito: "Solo corrige errores fonéticos obvios. Mantené verbos y estructura intactos." Tests cubren casos típicos. |
| Corrector inventa contactos que no existen en la libreta | high | Context pasa `knownContacts: List<String>` y prompt restringe correcciones de nombres a esa lista. |
| Latencia adicional rompe el SLA <3s del Plan C | medium | Corrector solo se invoca en casos ambiguos (no siempre). Timeout 1.5s. Plan C sigue siendo la ruta feliz para comandos del guion. |
| On-device recognizer no soporta es-AR en el device de demo | low | `RecognizerFactory` valida disponibilidad y cae a cloud. Smoke test obligatorio en device pre-demo. |
| Gemini devuelve transcript con quotes / markdown / texto extra | medium | Prompt instruye `responseMimeType: "application/json"` y schema `{"corrected": String}`. Parser tolera fallback a `rawTranscript`. |
</threat_model>

<title>SttCorrector + nueva entrada en PromptBuilder</title>
<action>
`SttCorrector.kt`:

```kotlin
class SttCorrector(private val llmClient: LlmClient) {
    suspend fun correct(raw: String, context: SttContext): String {
        val prompt = PromptBuilder.buildSttCorrectionPrompt(raw, context)
        return runCatching {
            withTimeout(1500) { llmClient.correctTranscript(prompt) }
        }.getOrDefault(raw)
    }
}

data class SttContext(val knownContacts: List<String>, val lastCommand: String?)
```

Extender `LlmClient` con `suspend fun correctTranscript(prompt: String): String` y su impl en `GeminiLlmClient` (modelo `gemini-2.5-flash` o `flash-lite`, config `temperature: 0`, `maxOutputTokens: 64`, `responseMimeType: "application/json"`).

- System prompt corto: *"Sos un corrector de transcripts en español argentino. Corregí errores fonéticos obvios manteniendo la intención. Si un nombre suena parecido a uno de los contactos conocidos, usá el contacto. Si no, dejá el nombre como está. NO inventes contactos."*

<action>

```kotlin
object RecognizerFactory {
    fun create(context: Context): SpeechRecognizer {
        val canOnDevice = SpeechRecognizer.isOnDeviceRecognitionAvailable(context)
            && Build.VERSION.SDK_INT >= 31
        return if (canOnDevice) {
            SpeechRecognizer.createOnDeviceSpeechRecognizer(context).also {
                Timber.tag("Beto-STT").d("Using on-device recognizer")
            }
        } else {
            SpeechRecognizer.createSpeechRecognizer(context).also {
                Timber.tag("Beto-STT").d("Using cloud-backed recognizer")
            }
        }
    }
}
```

Importante: `RecognizerIntent.EXTRA_LANGUAGE = "es-AR"` se mantiene; `EXTRA_PREFER_OFFLINE = true` setear cuando on-device.

<task id="03-02-T04" type="implement">
<title>Integración VoiceCaptureActivity con SttCorrector</title>
<action>

1. Tomar el mejor transcript + su confidence.
2. Si `confidence >= 0.6` y `DeterministicMatcher.matches(transcript)` → emitir `VoiceCaptured(transcript)` directo (ruta feliz Plan C).
