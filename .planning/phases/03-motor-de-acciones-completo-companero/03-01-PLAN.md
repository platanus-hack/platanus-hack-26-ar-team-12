# Plan 03-01: Deterministic Actions and Contact Resolution

## Goal
Extend the Phase 2 deterministic path from WhatsApp-only into the fixed demo action set: WhatsApp, calls, SMS drafts, and Maps. This gives Phase 3 a safe local baseline for demo phrases and for LLM outage fallback without guessing or substituting entities.

## Must Haves
<must_haves>
  <decision id="D-01">The exact demo phrases must work deterministically: "llama a mi hijo", "mandale un sms a Ana", and "abrime el mapa hasta la farmacia".</decision>
  <decision id="D-05">Fallback must never substitute action, contact, destination, app, or message. If Pedro is unresolved, Beto asks/fails warmly; it never calls hijo.</decision>
  <decision id="D-06">Contact resolution checks DemoContacts first, then Android Contacts.</decision>
  <decision id="D-07">Zero or multiple contact matches are not guessed.</decision>
  <constraint>SMS uses an SMS draft intent, not background SEND_SMS.</constraint>
  <constraint>Calls must respect CALL_PHONE permission state and fail warmly if unavailable.</constraint>
</must_haves>

## Threat Model
- Wrong contact: mitigated by demo-first resolver, explicit ambiguity handling, and no entity substitution.
- Unsafe SMS behavior: mitigated by SMS draft intent only.
- Permission crash: mitigated by typed action results and permission-state checks.
- Scope creep: mitigated by fixed active allow-list excluding `agentic_perform_action`.
