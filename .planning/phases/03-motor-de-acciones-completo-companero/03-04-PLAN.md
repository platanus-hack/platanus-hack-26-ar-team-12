# Plan 03-04: Companion Chat Sheet and Conversational Action Handoff

## Goal
Implement the long-press Companion experience as a simple voice-first chat sheet. Beto should feel conversational, Argentine, patient, and calm, while using the same safe fixed-action engine whenever the conversation leads to an action.

## Must Haves
<must_haves>
  <decision id="D-10">Long-press opens Companion and starts listening immediately.</decision>
  <decision id="D-11">Companion is voice-first with a minimal text bar and no giant microphone button.</decision>
  <decision id="D-12">Listening/recording animation uses vertical bars like "Hey Google".</decision>
  <decision id="D-13">The UI is a lightweight sheet with transcript, text bar, and listening state.</decision>
  <decision id="D-14">Tone is Argentine, simple/correct, patient/calm, conversational, and not boring.</decision>
  <decision id="D-15">Companion uses `gemini-2.5-flash-lite` with moderate temperature; action engine remains flash/temp0.</decision>
  <decision id="D-16">Companion can execute actions with confirmation.</decision>
  <decision id="D-17">The user should never see action mode vs companion mode.</decision>
  <decision id="D-18">Companion history is session-only ViewModel state.</decision>
  <deferred>Physical side button activation and persistent personal memory are deferred out of Phase 3.</deferred>
</must_haves>

## Threat Model
- Elder user confusion: mitigated by one Beto identity, immediate listening, minimal UI, and no technical mode labels.
- Accidental action from conversation: mitigated by confirmation before every Companion-initiated action.
- Privacy creep: mitigated by session-only state and no persistent personal-memory implementation in Phase 3.
- Model split regression: mitigated by explicit tests that Companion chat uses flash-lite while actions use flash/temp0.
