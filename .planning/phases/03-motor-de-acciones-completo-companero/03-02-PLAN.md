# Plan 03-02: LLM Router, Sanitizer, and Strict Tool Parsing

## Goal
Add the Gemini-backed action interpretation layer with privacy controls before any cloud call. The router must be deterministic in behavior, strict in parsing, and safe when Gemini returns malformed or out-of-scope output.

## Must Haves
<must_haves>
  <decision id="D-02">Gemini handles arbitrary phrasing; users should not notice whether local matcher or LLM handled the command.</decision>
  <decision id="D-03">The action model is `gemini-2.5-flash` with tool calling and `temperature: 0`.</decision>
  <decision id="D-04">Malformed Gemini output gets one repair retry, then falls back only when original semantics are preserved.</decision>
  <decision id="D-05">No fallback may substitute contact, destination, app, message, or action.</decision>
  <decision id="D-08">The Phase 3 allow-list is exactly `send_whatsapp`, `make_call`, `send_sms`, and `open_maps`.</decision>
  <decision id="D-09">If Gemini returns `agentic_perform_action`, the dispatcher rejects and logs it.</decision>
  <constraint>DNI, phone numbers, card numbers, and similar sensitive strings must be redacted before Firebase AI sees them.</constraint>
</must_haves>

## Threat Model
- PII leakage: mitigated by sanitizer-before-prompt and sanitized `Beto-LLM` logs.
- Wrong action from stochastic output: mitigated by `temperature: 0`, strict parsing, and fixed allow-list.
- Unsafe malformed repair: mitigated by one retry and no semantic substitution fallback.
- Agentic escalation: mitigated by explicit rejection of `agentic_perform_action`.
