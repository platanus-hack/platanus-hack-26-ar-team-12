# Phase 3 - UI Design Contract

> Visual and interaction contract for Modo Companero. Generated by `$gsd-ui-phase`, verified against the 6 UI quality dimensions.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Android Jetpack Compose for `CompanionActivity`; existing Android Views remain for floating overlay |
| Icon library | Existing drawable assets first; Material Icons only if Compose dependency is already added for this phase |
| Font | Android system sans-serif |

**Implementation boundary:** keep `OverlayBubble` as the existing 64dp Android View. Build the Companion surface as a translucent `CompanionActivity` with a sheet-like Compose layout, because `COMP-01` explicitly calls for Compose. Do not add a settings screen, onboarding screen, tab system, or separate mode selector.

**Primary screen:** a bottom sheet occupying 58-72% of screen height, anchored to the bottom, with a clear hint of the home/app underneath through a dimmed translucent backdrop.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon gaps, recording bar gaps |
| sm | 8dp | Compact internal spacing, chip/text-field padding |
| md | 16dp | Default spacing between chat rows and controls |
| lg | 24dp | Sheet side padding and top/bottom padding |
| xl | 32dp | Major separation between transcript and input area |
| 2xl | 48dp | Minimum bottom comfort area around voice/text controls |
| 3xl | 64dp | Reserved for full-screen fallback only |

Exceptions:
- Floating bubble remains 64dp with 8dp padding from the existing `overlay_bubble.xml`.
- Touch targets for text input send/voice controls must be at least 48dp.

## Typography

Use exactly these sizes and weights in Companion UI:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 16sp | 400 | 1.35 |
| Body | 22sp | 400 | 1.45 |
| Heading | 26sp | 600 | 1.25 |
| Display | 30sp | 600 | 1.2 |

Rules:
- Body chat text must be 22sp minimum to support older adults.
- Do not introduce additional font sizes in Phase 3.
- Use weight 600 only for speaker labels, sheet heading, and primary state text.
- Do not use negative letter spacing or viewport-scaled font sizes.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F8FAF7` | Companion sheet background and large calm surfaces |
| Secondary (30%) | `#FFFFFF` | Input field, user message bubbles, transcript surface |
| Accent (10%) | `#2F7D5C` | Active listening bars, primary send icon, focused input border, Beto speaker marker |
| Destructive | `#B3261E` | Fatal error text only; no destructive actions in Phase 3 UI |

Accent reserved for:
- Active vertical listening bars.
- Primary send action icon/button.
- Focus ring/border on the text bar.
- Small Beto speaker marker beside Beto responses.

Do not use accent for every button, every chat bubble, or decorative background fills. User messages use neutral secondary surfaces with dark text.

## Layout Contract

### Companion Sheet

- Opens on `AgentEvent.BubbleLongPressed`.
- Starts listening immediately after opening.
- Uses a bottom sheet visual: rounded top corners at 24dp, square bottom edge aligned to screen bottom.
- Sheet width is full screen on phones, with 16dp side inset on screens wider than 420dp.
- Max content width is 520dp for large devices.
- Backdrop is translucent black at 28% opacity; tapping outside closes only if no action confirmation is pending.

### Visual Hierarchy

1. Primary focal point: listening state row with vertical recording bars and the current state phrase.
2. Secondary focal point: latest Beto response in large 22sp text.
3. Tertiary focal point: minimal text bar at the bottom.

### Required Components

| Component | Contract |
|-----------|----------|
| State header | Shows `Beto` and one state phrase: `Te escucho`, `Estoy pensando`, `Te respondo`, or `No te entendi bien`. |
| Recording bars | 5 vertical bars, 4dp wide, rounded 2dp, heights animate between 10dp and 34dp while listening. |
| Transcript | Shows latest 3-5 turns only; newest at bottom; no scroll complexity needed for demo. |
| Text bar | Single-line field with placeholder `Escribile a Beto`; height 56dp; send icon target 48dp. |
| Action confirmation | Inline confirmation row for actions from Companion: action summary + `Hacerlo` + `Mejor no`. |

### Interaction Rules

- Long-press opens Companion and begins voice capture immediately.
- No giant microphone button.
- No visible "modo acciones" / "modo companero" labels.
- If the user asks for an action inside Companion, show/voice a confirmation before execution.
- If action execution starts, keep the UI language as one Beto: "Dale, lo hago" / "No encontre a Pedro".
- If voice capture fails, stay in the sheet and show a warm retry state, not a technical error.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Hacerlo` |
| Secondary action | `Mejor no` |
| Text input placeholder | `Escribile a Beto` |
| Empty state heading | `Te escucho` |
| Empty state body | `Decime que necesitás, tranquilo.` |
| Listening state | `Te escucho` |
| Thinking state | `Estoy pensando` |
| Speaking state | `Te respondo` |
| Warm retry state | `No te entendí bien. Probemos de nuevo, dale.` |
| Error state | `Uy, algo no salió bien. Probá de nuevo, dale.` |
| Contact missing | `No encontré a esa persona. ¿Me decís el nombre otra vez?` |
| Ambiguous contact | `Encontré más de una persona con ese nombre. ¿Cuál querés?` |
| Destructive confirmation | none |

Tone rules:
- Argentine, simple, correct, patient, calm, conversational, not boring.
- One short sentence for TTS feedback.
- No technical codes, stack traces, tool names, JSON names, or internal routing language in user-facing copy.

---

## Accessibility Contract

- All text must meet contrast ratio 4.5:1 or better against its background.
- Text bar and icon controls must expose content descriptions.
- Touch targets must be at least 48dp.
- Recording bars are decorative; they must not be the only indicator of listening. Pair them with `Te escucho`.
- The sheet must remain usable at Android font scale 1.3 without overlapping text.
- No flashing or high-frequency animation; recording bars animate gently with staggered durations between 420ms and 760ms.

---

## Animation Contract

| Element | Motion |
|---------|--------|
| Sheet enter | Slide up from bottom, 220ms, ease-out |
| Sheet exit | Slide down, 180ms, ease-in |
| Recording bars | Infinite staggered height animation while listening; stop and settle to 14dp when not listening |
| State changes | Crossfade text over 140ms |
| Confirmation row | Fade + expand over 160ms |

Do not animate layout in a way that moves the text bar while the user is typing.
