# Plan 03-03: Dispatcher Integration and Invisible Fallback

## Goal
Wire deterministic actions and the LLM router into the existing voice flow so Beto chooses the safest interpretation path, dispatches fixed Android actions, and hides local-vs-cloud routing from the elder user.

## Must Haves
<must_haves>
  <decision id="D-02">The user should not perceive a matcher-vs-Gemini mode switch.</decision>
  <decision id="D-04">Malformed Gemini gets one repair retry, then only same-semantics fallback.</decision>
  <decision id="D-05">Pedro can never turn into hijo; no fallback substitutes explicit entities.</decision>
  <decision id="D-09">`agentic_perform_action` is rejected/logged if returned.</decision>
  <decision id="D-17">There is one Beto experience, not visible action and companion modes.</decision>
  <constraint>The Phase 2 WhatsApp path must keep working.</constraint>
</must_haves>

## Threat Model
- User harm from wrong action/contact: mitigated by deterministic-first routing, explicit fallback blocking, and dispatcher rejection.
- Hidden technical failure: mitigated by warm TTS clarification and recoverable action outcomes.
- Regression of Phase 2: mitigated by routing WhatsApp through the new dispatcher with existing tests preserved.
- Privacy regression in logs: mitigated by reusing sanitized LLM log paths only.
