<objective>
Implementar activación de Beto por mantener pulsado el botón de volumen-down durante 2 segundos. La burbuja entra automáticamente en `LISTENING` y se lanza `VoiceCaptureActivity`. Funciona desde cualquier app o desde la home, mientras `BetoAccessibilityService` esté conectado. No requiere permisos adicionales.
</objective>

<user_story>
Como adulto mayor con tremor o dificultad para apuntar bien con el dedo, quiero poder activar a Beto manteniendo apretado un botón físico del celular en lugar de tener que apuntar la burbuja.
</user_story>

<must_haves>
- ACTIV-01: `BetoAccessibilityService` recibe `KeyEvent.KEYCODE_VOLUME_DOWN` y mide el tiempo entre `ACTION_DOWN` y `ACTION_UP`. Si ≥2000ms y no hubo otro key event en el medio (consistente long-press), dispara captura.
- Trigger emite `AgentCommand.StartListening(source = "volume_down_long_press")` en `AgentBus`.
- Mientras se mantiene presionado (entre 1.5-2.0s), Beto **NO** consume el evento. Si cumple los 2s, **sí** consume — para evitar que el sistema baje el volumen 100% mientras se mantiene.
- Configurable via `flagRequestFilterKeyEvents` en `accessibility_service_config.xml`.
- Si ya hay una captura activa (`AgentEvent.VoiceCaptureStarted` reciente), el long-press se ignora — evita doble trigger.
- Logs `Beto-Activation`: `VOL_DOWN_DOWN`, `VOL_DOWN_UP elapsedMs=...`, `ACTIVATION_TRIGGERED source=volume`.
</must_haves>

<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| Beto consume volume keys de apps (música, video) | high | Solo consume el evento si se completaron los 2s con long-press. Eventos cortos (<2s) pasan al sistema normalmente. |
| Long-press mientras Beto ya está escuchando dispara dos veces | medium | Estado interno del activator chequea `isCurrentlyListening` antes de emitir. |
| Bajada de volumen incremental durante el long-press deja audio mute | high | Mientras se acumula tiempo, NO consumimos el evento → sistema baja volumen progresivamente como esperado. Solo al cumplir 2s consumimos y restauramos volumen al previous (opcional). |
| `flagRequestFilterKeyEvents` requiere user re-acepte el AccessibilityService | medium | Documentar en setup. Banner en MainActivity informando que es necesario. |
| User confunde con shortcut de captura de pantalla (vol-down + power) | low | Si vol-down se combina con power, Android maneja screenshot — no llega como key event independiente típicamente. Aceptable. |
</threat_model>

<task id="06-01-T03" type="implement">
<title>VolumeKeyActivator</title>
<action>
`activation/VolumeKeyActivator.kt`:

```kotlin
class VolumeKeyActivator(
    private val isListening: () -> Boolean,
    private val onTrigger: () -> Unit,
    private val nowMs: () -> Long = { SystemClock.elapsedRealtime() },
) {
    private var pressDownTime: Long = 0L
    private var triggered: Boolean = false

    fun handleKeyEvent(event: KeyEvent): Boolean {
        if (event.keyCode != KeyEvent.KEYCODE_VOLUME_DOWN) return false

        return when (event.action) {
            KeyEvent.ACTION_DOWN -> handleDown(event)
            KeyEvent.ACTION_UP -> handleUp(event)
            else -> false
        }
    }

    private fun handleDown(event: KeyEvent): Boolean {
        if (event.repeatCount == 0) {
            pressDownTime = nowMs()
            triggered = false
            Timber.tag("Beto-Activation").d("VOL_DOWN_DOWN")
        }
        // Long-press detection: si supera el threshold y aún está apretado
        val elapsed = nowMs() - pressDownTime
        if (elapsed >= THRESHOLD_MS && !triggered && !isListening()) {
            triggered = true
            Timber.tag("Beto-Activation").d("ACTIVATION_TRIGGERED source=volume elapsed=$elapsed")
            onTrigger()
            return true // consume
        }
        return false
    }

    private fun handleUp(event: KeyEvent): Boolean {
        val elapsed = nowMs() - pressDownTime
        Timber.tag("Beto-Activation").d("VOL_DOWN_UP elapsedMs=$elapsed triggered=$triggered")
        val wasTriggered = triggered
        triggered = false
        pressDownTime = 0L
        return wasTriggered // consume si triggeró
    }

    companion object { const val THRESHOLD_MS = 2000L }
}
```

<title>Wire al AccessibilityService</title>
<action>
`BetoAccessibilityService` override:

```kotlin
private val volumeActivator = VolumeKeyActivator(
    isListening = { /* check AgentBus state */ },
    onTrigger = {
        AgentBus.emit(AgentCommand.StartListening("volume_down_long_press"))
        startVoiceCaptureActivity()
    }
)

override fun onKeyEvent(event: KeyEvent): Boolean {
    return volumeActivator.handleKeyEvent(event)
}
```
