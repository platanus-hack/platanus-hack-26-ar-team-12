<objective>
Harden the Phase 2 vertical slice so it handles empty STT, ambiguous close matches, and WhatsApp launch failures with exactly one retry/clarification and warm Android TTS phrases.
</objective>

<user_story>
As an older adult, I want Beto to ask once when it did not hear enough and to fail warmly when WhatsApp cannot open, so that mistakes do not leave me stuck or confused.
</user_story>

<must_haves>
- D-02: One warm retry on empty/unusable STT, then final failure.
- D-07: Matcher supports multiple verbs/connectors in Spanish/Argentine variants.
- D-08: Missing contact/message gets one short voice clarification, then complete or fail warmly.
- D-12: WhatsApp intent failure gets TTS failure only and stops.
- D-15: WhatsApp failure phrase is exactly "No pude abrir WhatsApp. Probemos de nuevo en un ratito, dale."
- D-16: STT/matcher final failure phrase is exactly "Perdon, no te entendi bien. Probemos de nuevo, dale."
- D-17: Android native TTS only; no ElevenLabs.
</must_haves>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Voice retry loop traps the user | high | Controller stores retry/clarification counts and caps each at one. |
| Ambiguous command sends/prepares wrong message | high | Missing contact/message asks one focused clarification instead of inferring defaults. |
| Error exposes technical details to older adult | medium | All failure branches speak locked warm phrases without exception messages. |
| ElevenLabs/live network sneaks into offline path | medium | Acceptance criteria grep for absence of `ElevenLabs` and HTTP voice code in Phase 2 controller. |
</threat_model>

<action>
Create `PlanCController.kt` or equivalent service-owned controller with explicit state:

- `sttRetryUsed: Boolean`
- `clarificationUsed: Boolean`
- optional pending partial match for missing contact/message.

Implement behavior:

- Empty/cancel/unusable STT result:
  - if `sttRetryUsed == false`, speak exactly `No te escuche bien. Probemos de nuevo, dale.`, set `sttRetryUsed = true`, and launch voice capture again.
  - else speak exactly `Perdon, no te entendi bien. Probemos de nuevo, dale.` and reset state.
- `NeedsContact`: speak exactly `A quien le aviso?`, set `clarificationUsed = true`, capture one follow-up, merge contact answer into pending message.
- `NeedsMessage`: speak exactly `Que queres que le diga?`, set `clarificationUsed = true`, capture one follow-up, merge message answer into pending contact.
- If clarification was already used and the command still does not match, speak exactly `Perdon, no te entendi bien. Probemos de nuevo, dale.` and reset state.
- Do not allow more than one STT retry or more than one clarification per user command.
</action>
