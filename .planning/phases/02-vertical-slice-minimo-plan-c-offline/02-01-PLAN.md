<objective>
Deliver the first real Phase 2 vertical slice: a user taps Beto's bubble, speaks the demo command, the app captures one final transcript, deterministically matches "nieto" and message text, opens regular WhatsApp with the message pre-filled, and speaks the locked confirmation/success phrases.
</objective>

<user_story>
As an older adult, I want to tap Beto and say "mandale a mi nieto que ya llegue", so that WhatsApp opens with the correct message ready without me navigating the phone.
</user_story>

<must_haves>
- D-01: Use transparent `VoiceCaptureActivity` hosting Android `RecognizerIntent.ACTION_RECOGNIZE_SPEECH` with `es-AR`.
- D-03: Emit only the best final transcript via `AgentBus`.
- D-04: Log tap -> final transcript elapsed milliseconds.
- D-05: `DeterministicMatcher` covers the demo phrase family, not arbitrary natural language.
- D-06: `DemoContacts.kt` resolves `nieto`, `mi nieto`, and the real demo contact name to one E.164 number.
- D-09: Open WhatsApp pre-filled only; do not auto-send.
- D-10: Target strict package `com.whatsapp`.
- D-11: Treat successful intent launch as success.
- D-13: Pre-launch phrase is exactly "Abro WhatsApp con el mensaje para tu nieto."
- D-14: Success phrase is exactly "Listo, te deje el mensaje preparado."
</must_haves>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Accidental message auto-send without user review | high | This plan only pre-fills WhatsApp and never performs Accessibility click/send automation. |
| Wrong contact receives prefilled message | high | `DemoContacts` is hardcoded for the demo aliases and unit-tested; no fuzzy contact lookup in this plan. |
| Sensitive transcript data logged | medium | Logs may include demo transcript only; do not log real phone numbers except sanitized/digits-only test values. |
| App chooser opens WhatsApp Business or another app | medium | Intent uses `setPackage("com.whatsapp")`; missing package is handled in Plan 02-02. |
</threat_model>
