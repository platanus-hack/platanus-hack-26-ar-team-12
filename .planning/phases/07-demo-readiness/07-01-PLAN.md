<threat_model>
| Threat | Severity | Mitigation |
|---|---|---|
| AccessibilityService se desactiva entre install y demo | high | Banner persistente en MainActivity si está desactivado. Pre-flight check al boot. Atajo a Settings disponible. |
| Voz TTS es-AR no instalada en device | high | Verificación en checklist físico. Backup: device pre-configurado con voz instalada. |
| Hotspot del dev se queda sin datos | medium | Verificar cuota en checklist. Backup: segundo dev como hotspot alternativo. |
| Wi-Fi del venue impone firewall que bloquea Firebase | medium | Hotspot personal evita el problema. Test pre-demo. |
| App actualizada accidentalmente <4h antes | high | Checklist explícito: "APK no actualizado en 4hs". Política de freeze del repo en branch `demo-freeze`. |
| Permiso runtime se reseteó (READ_CONTACTS / READ_SMS) | medium | Pre-flight check + atajo a App Info. |
| Voz masculina no disponible — TTS suena femenina | high | Verificación temprana en `docs/04-VOICE-AB.md`. Si no se puede asegurar masculina, ajustar el script para ser ambiguo de género (Beto puede ser nombre cualquier voz). |
| Video respaldo no carga porque está en cloud y no hay red | high | Copia local en el device del presentador + en laptop del dev. |
| Pre-flight crash o timeout silencioso | medium | Implementar logs visibles en pantalla durante boot durante la última build (debug overlay). |
</threat_model>

<task id="07-01-T04" type="ops">
<title>Guion de demo (07-DEMO-SCRIPT.md)</title>
<action>
Crear `docs/07-DEMO-SCRIPT.md` con el guion exacto de 3-5 min:

**Apertura (15s)**: contexto del problema (adultos mayores + brecha digital).

**Demo del Motor de Acciones (60s)**:
1. *"Beto, mandale a mi nieto que ya llegué"* → Beto pregunta "¿Quién es tu nieto?" (porque memoria está vacía al boot del demo) → user dice "Pedro" → Beto guarda y manda WhatsApp.
2. *"Beto, llamá a mi hijo"* → memoria ya tiene un alias previo (pre-seeded) → Beto llama directo.
3. *"Beto, mandale un mensaje a la Dra López"* → primera vez → Beto pregunta "¿por WhatsApp, SMS o llamada?" → user dice WhatsApp → Beto guarda + manda.

**Demo del Modo Compañero (45s)**:
- Long-press en burbuja → sheet abre.
- *"Beto, ¿cómo estás?"* → respuesta cálida.
- *"Me gusta el tango"* → Beto responde + tarjeta "¿Querés que me acuerde?" → user toca Sí.

**Demo del Modo Guía con gestos (60s)**:
- *"Beto, ¿cómo mando un audio por WhatsApp?"* → Beto abre WhatsApp + flecha animada sobre el botón micrófono + voz explicando paso a paso.

**Demo del Anti-fraude (si Phase 5 entró, 30s)**:
- Compartir SMS de fraude pre-cargado vía share-target → Beto analiza y veredicta.

**Cierre (15s)**: pitch del impact + roadmap (computadora, anti-fraude proactivo, etc.).

Documentar tiempos por bloque, transiciones, frases EXACTAS que dirá el presentador.

**Recuperación de errores documentada**:
- Si STT no entiende → repetir más despacio.
- Si LLM tarda → frase puente *"Dame un segundo"*.
- Si Intent falla → switch al hot-spare phone.
- Si todo falla → cortar al video pre-grabado.
</action>
