## Pitfalls encontrados

1. **`temperature()` deprecated en Haiku 4.5.** El SDK loguea: *"Models released after Claude Opus 4.6 do not support setting temperature. A value of 1.0 will be accepted; all other values will be rejected with 400."* Removí todas las llamadas a `.temperature(...)`. Para variar tono entre el motor de acciones (estricto JSON) y el Compañero (chat natural), uso únicamente system prompts y `maxTokens`.

2. **Conflict META-INF/DEPENDENCIES.** `anthropic-java` trae `httpclient5` + `httpcore5-h2` + `httpcore5` transitivos, los tres con su propio `META-INF/DEPENDENCIES`. AGP falla en `mergeDebugJavaResource`. Resuelto con un `packaging.resources.excludes` (patrón estándar Android, no afecta runtime).

3. **JDK environment.** El daemon Gradle local quedó cacheado con JDK 25.0.1, que el Kotlin compiler embebido en Gradle 8.10 no parsea (`IllegalArgumentException: 25.0.1`). Solución de sesión: `./gradlew --stop` + `JAVA_HOME=/opt/homebrew/opt/openjdk@21/...`. **Acción para el equipo:** mover `org.gradle.java.home` desde `local.properties` (no es honrado allí) a `gradle.properties`, o documentar el `JAVA_HOME` en el README. Lo dejo fuera del scope de este quick task — no es regresión introducida por la migración, ya bloqueaba el build pre-cambio.

4. **DSL `java.util.Properties()`.** En `build.gradle.kts` el accessor `java` clashea con el DSL. Resuelto con `import java.util.Properties` al tope.

## Setup requerido por el dev

Agregar a `android/local.properties` (gitignored):
```
anthropic.api.key=sk-ant-api03-...
```

Sin la key, `BuildConfig.ANTHROPIC_API_KEY` queda vacío y todas las llamadas devolverán string vacío con `LLM_GENERATE_FAILED` en logs. La app no crashea — el ActionRouter cae a fallbacks deterministicos / matchers.
